# **5. How to Ask Codex to Work Fast**

Here are powerful prompt patterns for real-world work.

---

## **5.1 Pattern: “Do the whole task in steps.”**

```text
Read the project and complete these steps:

1. Understand the current structure.
2. Suggest improvements.
3. Generate updated folder structure.
4. Generate code for each part.
5. Add explanation + migration plan.
```

---

## **5.2 Pattern: “Fix and upgrade at the same time.”**

```text
Debug this file and also refactor it.
Make code readable and add docstrings.
Do not change behaviour.
```

---

## **5.3 Pattern: “Add new features without breaking old code.”**

```text
Add the following feature.
Do not modify existing API signatures.
If needed, extend models using new fields.
Provide Alembic migration script.
```

---

## **5.4 Pattern: “Read my project and tell me what’s wrong.”**

```text
List 10 problems in this codebase.
Explain why each is a problem.
Show improved code or design.
```

This saves huge time during reviews.

---

## **5.5 Pattern: “Convert business logic into code.”**

Clients often describe features informally.

You can tell Codex:

```text
Convert this feature description into proper API design + database schema + code.
```

Codex turns business text → working code.
