# **Chapter 10. Form Basics – Collecting Data from Users**

---

## **1. Simple Introduction**

Forms are the **front desk** of your application.
Whenever your app needs information from a user — name, age, gender, choices — you use a **form**.

Forms are made of simple building blocks such as:

* **Text boxes** → for typing words or numbers
* **Dropdowns** → for selecting one item from a list
* **Radio buttons** → for selecting exactly one option
* **Checkboxes** → for selecting one or many options

Once a user fills these, you can send the data to your **backend API**, save it in a database, or show something on screen.

---

## **2. Real-Life Analogy**

Imagine you go to a **clinic** for the first time.
The receptionist gives you a **New Patient Registration form**:

* Your name
* Your phone
* Your age
* Your gender
* Any past medical issues

This is exactly what a frontend form does — it collects information so the system can work properly.

---

## **3. Key Concepts (Explained Gently)**

### **Text Box**

A simple place where users type things like name, address, age.

### **Dropdown**

Useful when only a few choices are allowed — like selecting **gender**, **blood group**, or **doctor’s department**.

### **Radio Buttons**

Let users select **only one** option — like “Male/Female/Other”.

### **Checkboxes**

Let users select **multiple** options — like “Diabetes”, “Asthma”, “Hypertension”.

### **Required vs Optional Fields**

* **Required**: Must be filled, or the system cannot continue (e.g., patient name).
* **Optional**: Can be left blank (e.g., alternative contact number).

---

## **4. Step-by-Step: How Forms Work**

### **Step 1 – Show Inputs**

Your UI displays text boxes, dropdowns, radio buttons, etc.

### **Step 2 – User Fills the Data**

### **Step 3 – Validate**

Check required fields:

* Name must not be empty
* Age must be a number

### **Step 4 – Submit**

Send the data to your backend API in JSON.

### **Step 5 – Show Result**

Display confirmation or errors.

## **5. Example UI Code (NiceGUI – very simple)**

```python
from nicegui import ui

def submit():
    ui.notify(f"Saved: {name.value}, {age.value}, {gender.value}, {conditions.value}")

with ui.card().classes('p-4'):
    ui.label('New Patient Registration')

    name = ui.input('Full Name *')         # required
    age = ui.input('Age *').props('type=number')

    gender = ui.select(
        ['Male', 'Female', 'Other'],
        label='Gender *'
    )

    conditions = ui.checkbox_group(
        ['Diabetes', 'Hypertension', 'Asthma'],
        label='Existing Conditions (optional)'
    )

    ui.button('Submit', on_click=submit)

ui.run()
```

This simple form collects patient registration data.

## **6. Backend API Example (Very Simple)**

Imagine your backend has this endpoint:

```
POST /patients/new
```

And the JSON looks like:

```json
{
  "name": "Rahul Das",
  "age": 45,
  "gender": "Male",
  "conditions": ["Diabetes"]
}
```

Your frontend will send this JSON to the backend when the user clicks **Submit**.
