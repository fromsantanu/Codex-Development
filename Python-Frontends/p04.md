## **4. Step-by-Step: How Codex Helps You**

---

### **A) Generating UI Code Skeletons**

You simply describe the UI you want.

Example request to Codex:

> “Make a Streamlit page with name, age, and symptoms fields and a submit button.”

Codex will instantly generate:

```python
import streamlit as st

st.title("Patient Check Form")

name = st.text_input("Name")
age = st.number_input("Age", min_value=0)
symptoms = st.text_area("Symptoms")

if st.button("Submit"):
    st.success("Data submitted!")
```

This saves time and ensures a clean starting point.

---

### **B) Creating and Wiring API Calls to Backend URLs**

Codex can wire your frontend to your backend easily.

Example:

> “Add a POST request to [http://localhost:8000/check](http://localhost:8000/check) using the form data.”

Codex might generate:

```python
import requests

if st.button("Submit"):
    response = requests.post(
        "http://localhost:8000/check",
        json={"name": name, "age": age, "symptoms": symptoms}
    )
    data = response.json()
    st.write(data)
```

This makes your UI fully functional.

---

### **C) Helping with Layout, Tabs, Columns, Cards**

If your layout feels messy, you can ask:

> “Make this Streamlit layout cleaner using columns.”

Codex generates:

```python
col1, col2 = st.columns(2)

with col1:
    name = st.text_input("Name")

with col2:
    age = st.number_input("Age")

symptoms = st.text_area("Symptoms")
```

Looks more professional instantly.

---

### **D) State Handling (Remembering Data)**

Codex can insert session state variables for you.

Example instruction:

> “Store the last response in session state.”

Codex will produce:

```python
if "last_result" not in st.session_state:
    st.session_state.last_result = None

if st.button("Submit"):
    response = requests.post(...)
    st.session_state.last_result = response.json()

st.write(st.session_state.last_result)
```

## **6. Example: Before and After Refactoring**

### **Before**

```python
st.title("Page")
a = st.text_input("n")
b = st.text_input("c")
if st.button("ok"):
    r = requests.post("http://x/y", json={"a": a, "b": b})
    st.write(r.json())
```

### **After (Codex Refactored)**

```python
import streamlit as st
import requests

st.title("User Information Form")

name = st.text_input("Name")
city = st.text_input("City")

def submit_data(name, city):
    response = requests.post(
        "http://localhost:8000/save",
        json={"name": name, "city": city}
    )
    return response.json()

if st.button("Submit"):
    if not name or not city:
        st.error("Please fill all fields.")
    else:
        result = submit_data(name, city)
        st.success(result)
```

Cleaner, easier, and more structured.
